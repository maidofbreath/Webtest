User Experience Design

When thinking about web design, you should take into account the UI versus the UX. UI Encompasses the User Interface, what the client will see when they navigate your web page. This ranges from the size of the buttons they encounter to the color of the text they read. UI is a part of the broader UX, or User Experience. UX considers not just how the web page looks, but how well it functions and how intuitive it is to traverse. According to this Figma article, when designing the UX, you should follow these five steps:

Consumer Research

Research what the average user of your website would need or expect out of it. It would be inappropriate for an eye doctor's page to have very small text in clashing colors.

Information Architecture

Once you feel research is complete, you should make a blueprint outling navigation, content heirarchy, features, and interactions you envision with your website. It should be visual focused, being sketched or mapped out before you jump into coding.

Create Prototypes

Once satisfied with your blueprint, you should workshop the bare bones version of your design. This makes it easy for you and others to find out what is missing or is superfluous, like a rough draft on an essay.

Debug and Test

A fully fleshed code is never without bugs, so make sure to take your time building and ensure the function of your website is as strong as you can make it. This includes anticipating actions that may cause errors to occur.

Continually Update

Even though the majority of the work is done, you should still periodically check and update your website. As time moves, technology, needs, and standards will develop and it is your job to reflect these changes on your website.

User Interface Design

User Interface design follows many of the same principles as general graphic design. When designing the visual aspects of a website, your design should not only look appealing, but aid in the function of the UX. Websites, first and foremost, are a means of communication. It should be your goal to make this communication go as smoothly as possible, and a bad visual design will bring down any base code, regardless of how functional it is. Concrete CMS breaks down User Interface design into 7 principles:

Unity

Make sure everything looks like it belongs on the same website.

Balance

Flow is important for aesthetic, don't make your page too lumpy or lopsided.

Heirarchy

Make sure important information is easily spotted or findable, with minor details taking up less space.

Contrast

While everything should look like it belongs, don't make it look too bland. Contrast is especially useful for drawing user's attention to details of high importance.

Emphasis

Much like with contrast, you should make things stand out when necessary.

Scale

Continuing with the contrast theme, utilizing size difference can be a powerful tool for drawing user focus.

Repition

Lastly, while you don't want everything to be bland, you do want everything to be consistent. An inconsistent design can confuse users or put a bad taste in their mouths.

Bad Webpage Design

As captivating as the website is, Cameron's World is in no way intuitive nor clear. While this is clearly a hobby site, a layout like this would alienate anyone who dared venture onto, say, an amusement park's website to buy tickets. The lack of labels, the clickable image links blending in with unlinked images and the background, links that lead no where, and lack of cohesion makes it very difficult to use. And while not meant to be taken seriously, User Inyerface is a game in which the player must navigate an intentionally infuiating UI in order to win. It's a neat way to show the importance of good design when making a functional website, as it breaks every rule from heirarchy to color theory and everything in between in the spirit of challenge.
